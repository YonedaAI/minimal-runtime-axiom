# The Minimal Runtime Axiom - Knowledge Base

## Core Axiom

The Minimal Runtime Axiom states: a decision d reaches runtime if and only if no static witness can be constructed for it. Formally:

R(P) = D(P) \ C(P), where C(P) = { d in D(P) | there exists a static witness w_d }

A static witness w_d = (tau, pi, sigma) where tau is a type encoding the invariant, pi is a proof term (inhabitation of tau), and sigma is a staging certificate that pi is constructible before execution.

## Corollaries

1. **Runtime Irreducibility**: d is legitimately runtime iff d depends on omega-data (user input, I/O, time, nondeterminism)
2. **The Duality**: |R(P)| is proportional to the epistemic deficit of the type system
3. **Yoneda Correspondence**: Runtime inspection is isomorphic to Yoneda failure

## Related Work

- MetaOCaml (Kiselyov): multi-stage programming, staging certificates
- Idris/Agda: dependent types as static witnesses
- Rust: monomorphization, zero-cost abstractions, ownership as static witness
- GHC rewrite rules: compile-time decision transfer
- Partial evaluation (Jones, Gomard, Sestoft): binding-time analysis
- Futamura projections: staging hierarchy
- Curry-Howard correspondence: proofs as programs
- Yoneda lemma: objects determined by relationships
- JAPL: effect systems, linearity as compile-time transfer mechanisms
- Staging calculus (Davies, Pfenning): modal logic for staged computation
